
# 🌌 Руководство по пакетной обработке в JAVELIN

Данный набор скриптов предназначен для автоматизированной подготовки фотометрических данных, пакетного запуска MCMC-моделирования в программе JAVELIN и последующей визуализации результатов.

Репозиторий исходного проекта JAVELIN:  https://github.com/nye17/javelin

**Стек технологий:** `Python 2.7` | `Ubuntu 18.04` | `Tkinter` | `Matplotlib`

### 🔄 Краткий обзор рабочего процесса

Весь пайплайн состоит из 4 логических шагов:

1. 🧹 **`preparation.py`** — очистка сырых данных и создание структуры папок.
2. ⚙️ **`run-javelin.py`** — пакетный запуск вычислений (генерация цепей Маркова).
3. 🎨 **`hist-tuner.py`** — графический интерфейс (GUI) для настройки внешнего вида гистограмм.
4. 📊 **`chains2hist.py`** — пакетная генерация графиков (PNG) для всех файлов.

---

## 🐳 Использование Docker (Windows 10+)

Так как скрипты требуют устаревшего **Python 2.7** и графического интерфейса, удобнее всего запускать их через Docker-контейнер.

### 1. Подготовка файловой системы

Создайте базовую папку (например, `C:\Javelin\`), где будут лежать Docker-файлы. Внутри создайте папку `mydata` — это будет наша **рабочая среда**, которую мы “пробросим” внутрь контейнера.

```text
📁 C:\Javelin\
 ├── 🐳 Dockerfile (и другие файлы образа)
 └── 📁 mydata\                   <-- Эту папку будет видеть Docker
      └── 📁 object1\             <-- Папка конкретного объекта
           └── 📁 scripts\        <-- Ваши python-скрипты
```

### 2. Запуск контейнера

Откройте терминал (cmd/PowerShell), перейдите в папку `C:\Javelin\` и выполните команду:

```bash
cd C:\Javelin\
docker run -it --rm -v "%cd%\mydata:/workspace" -w /workspace/object1/scripts -e DISPLAY=host.docker.internal:0.0 ub18-jav:latest
```

**Разбор команды по частям:**

- `-it` — интерактивный режим (чтобы вы могли вводить команды в терминале контейнера).
- `--rm` — автоматически удалить контейнер после закрытия (не засоряет систему).
- `-v "%cd%\mydata:/workspace"` — связывает вашу папку `mydata` с папкой `/workspace` внутри Docker. _Все изменения файлов сохранятся на вашем ПК._
- `-w /workspace/object1/scripts` — сразу открывает нужную папку при старте контейнера.
- `-e DISPLAY=...` — позволяет Docker “отправлять” графические окна (для `hist-tuner.py`) на ваш монитор.
    
    > 💡 **Важно для Windows:** Чтобы GUI открылось, на вашем ПК должен быть запущен X-сервер (например, **VcXsrv** или **Xming**), настроенный на прием подключений (опция _Disable access control_).

## 📂 Шаг 0: Расположение исходных данных

Подразумевается, что вы работаете с одним объектом в рамках одной директории (например, `object1`). Создайте папку `light_curves` на одном уровне со скриптами и положите туда ваши сырые `.txt` файлы.

```text
📁 object1/
 ├── 📁 scripts/               <-- Вы находитесь здесь
 │    ├── preparation.py
 │    └── ...
 └── 📁 light_curves/          <-- Сюда кладем исходные данные (.txt)
      ├── my_object_cont.txt
      └── my_object_line.txt
```

> 📌 **Требования к `.txt` файлам:**
> 
> - **3 колонки:** MJD, Поток, Ошибка потока (через пробел/таб).
> - **Комментарии:** строки с `#` в начале игнорируются.
> - **Имена:** Файлы континуума и линии должны иметь суффиксы `_cont` и `_line`.

---

## 🧹 Шаг 1: Подготовка данных (`preparation.py`)

Находясь в папке `scripts`, запустите первичную обработку:

```bash
python2 preparation.py
```

**Что произойдет автоматически:**

1. Создадутся нужные папки (`jav_data`, `results`, `light_curves/_versions).
2. Сырые `.txt` очистятся от мусора и сохранятся как **`.dat`** в папку `jav_data/`.
3. В папке `light_curves/` появится базовый файл настроек **`start1.cfg`**.

---

## ⚙️ Шаг 2: Настройка параметров MCMC

Откройте сгенерированный файл `light_curves/start1.cfg` (можно создать его копии для разных прогонов: `runA.cfg`, `runB.cfg`).

```ini
[paths]
# Маски поиска сконвертированных кривых блеска (можно указать точные имена)
cont_pattern = ../jav_data/*_cont*.dat
line_pattern = ../jav_data/*_line*.dat

# Папка для сохранения результатов (цепей Маркова)
chains_path = ../jav_data/chains_run1/

# Путь лог-файла 
log_path = ../jav_data/logs/run1.log

[mcmc]
n_walkers = 100       # Количество "ходоков"
n_burn = 500          # Период "разогрева" (burn-in)
n_chain = 500         # Длина цепи
lag_limit_min = 0     # Минимальный предел запаздывания (lag)
lag_limit_max = 10    # Максимальный предел запаздывания
n_iter = 50           # Количество генерируемых файлов (.jav)
```

> ⚠️ **Критически важно:** Если вы запускаете несколько разных `.cfg` файлов, обязательно меняйте в них пути `chains_path` и `log_path`, иначе новые расчеты перезапишут старые!

---

## 🚀 Шаг 3: Пакетный запуск вычислений (`run-javelin.py`)

Когда `.dat` файлы готовы, а `.cfg` настроены, запускаем расчет:

```bash
python2 run-javelin.py
```

- Скрипт сам найдет все `.cfg` файлы и запустит их по очереди.
- Вы увидите **шкалу прогресса** в терминале.
- Весь технический текст (спам) от JAVELIN скрыт и пишется в `.log` файл.
- Итог работы: папка `chains_run1` заполнится файлами **`.jav`** (цепи Маркова).

---

## 🎨 Шаг 4: Настройка графиков (`hist-tuner.py`)

Для подбора красивого вида гистограмм используем графический интерфейс:

```bash
python2 hist-tuner.py
```

1. **Загрузка:** Нажмите `Open chain` и выберите один репрезентативный `.jav` файл.
2. **Параметры:**
    - `column` = `3` (колонка задержки линии / lag).
    - Настройте `x_min`, `x_max`, `bins` (детализация).
    - `lag_peak` — позиция вертикальной красной линии (пик).
3. **Предпросмотр:** Нажмите кнопку `Plot hist`. Появится график. Закройте его, подправьте цифры и повторите, пока не добьетесь идеала.
4. **Стиль:** Можно задать цвета в HEX: `hist_color` (например `#601fb4`) и `line_color` (например `#ff0000`).
5. **Сохранение:** Нажмите  кнопку `Save config`. Настройки запишутся в файл **`hist.ini`**.

> 🧩 **Общая отрисовка:** тюнер и пакетный экспортёр используют один модуль
> (`histlib.py`) — и для схемы `hist.ini`, и для кода построения графика,
> поэтому предпросмотр гарантированно совпадает с экспортируемыми PNG. Цвета,
> заданные в GUI, теперь сохраняются в `[Style]` и переиспользуются
> `chains2hist.py` (старые `hist.ini` с секцией `[Data] column_number` тоже
> читаются).
>
> Секции `hist.ini`: `[Path]` (data_dir, file_pattern, output_dir),
> `[Plot]` (column, bins, x_min/x_max/y_min/y_max, dpi — пределы осей могут быть
> дробными), `[Annotations]` (подписи, lag_peak, размеры шрифтов),
> `[Style]` (yaxis_right, hist_color, line_color).

---

## 📊 Шаг 5: Пакетная генерация гистограмм (`chains2hist.py`)

Когда внешний вид настроен в `hist.ini`, генерируем картинки для всех файлов махом:

```bash
python2 chains2hist.py
```

Скрипт прочитает настройки, найдет все `.jav` файлы, отсортирует их (1, 2… 10) и сохранит высококачественные `.png` графики в папку `results/run1/`.

## 🌳 Итоговая структура проекта

После полного цикла ваша рабочая папка примет следующий вид:

```text
📁 object1/
 ├── 📁 scripts/
 │    ├── preparation.py
 │    ├── run-javelin.py
 │    ├── hist-tuner.py
 │    ├── chains2hist.py
 │    ├── histlib.py            <-- Общая схема hist.ini + построение графиков
 │    └── hist.ini              <-- Ваш сохраненный пресет графиков
 │
 ├── 📁 light_curves/           
 │    ├── 📁 _versions/         <-- Резервные копии сырых данных
 │    ├── start1.cfg            <-- Ваши конфиги MCMC
 │    ├── my_obj_cont.txt
 │    └── my_obj_line.txt
 │
 ├── 📁 jav_data/               
 │    ├── my_obj_cont.dat       <-- Сковертированные данные
 │    ├── my_obj_line.dat
 │    ├── 📁 chains_run1/       <-- Сгенерированные цепи (.jav)
 │    │    └── javChain_1.jav   
 │    └── 📁 logs/              <-- Логи вычислений
 │         └── run1.log
 │
 └── 📁 results/                
      └── 📁 run1/              <-- Итоговые графики!
           ├── jav_Simulation_lag_chain1.png
           └── jav_Simulation_lag_chain2.png
```

## 🛠 Устранение частых проблем

|Проблема / Ошибка|Решение|
|---|---|
|**`No module named ConfigParser`**<br>или синтаксические ошибки|Скрипты написаны под Python 2. Убедитесь, что запускаете их строго через `python2`, а не `python3`.|
|**`No files found matching pattern...`**|**При расчетах:** Проверьте правильность путей `cont_pattern` / `line_pattern` в вашем `.cfg` файле.<br>**При отрисовке:** В GUI-тюнере проверьте поля `data_dir` и `file_pattern`, затем нажмите _Save config_.|
|**`Failed to process...`** (в `preparation.py`)|Проблема с правами доступа. Выполните команду `chmod -R 755 .` в корне проекта (или проверьте права папки на хост-машине).|
|**`No module named Tkinter`**|Не установлена библиотека для GUI. Установите её в Ubuntu: `sudo apt-get install python-tk`. _(В Docker-образе она уже должна быть)._|
|**`ValueError: could not convert string to float`**|При предпросмотре графика в GUI в цифровое поле попала буква или пробел. Десятичные дроби пишите строго через точку (например: `3.5`, а не `3,5`).|
|**Графическое окно не открывается (в Docker)**|Убедитесь, что у вас запущен VcXsrv/Xming на Windows, а в Docker передан верный IP (переменная `DISPLAY`).|
